# App Shell Recipe

Buildpad has **two shells for two different jobs** ‚Äî don't confuse them:

- **`AuthenticatedShell`** ‚Äî the generic app chrome (glass header, sidebar
  navigation, profile menu) that `bootstrap` wires around your authenticated
  pages by default. Use it for your own screens ‚Äî dashboards, settings,
  anything custom. Jump to
  [A standalone shell for your own pages](#a-standalone-shell-for-your-own-pages).
- **`ContentLayout`** (with `ContentNavigation`, `CollectionList`, and
  `CollectionForm`) ‚Äî a schema-driven **content-management** recipe: a
  collapsible collection sidebar, a header bar with breadcrumbs and actions, a
  paginated list view, and a create/edit form, all driven by your DaaS schema.
  **The rest of this page walks through this recipe.**

![ContentLayout content-management screen ‚Äî collection sidebar and list view (from the Collections Storybook)](/app-shell-list.png)

## The pieces

| Piece | Package | Role |
|---|---|---|
| `ContentLayout` | `@buildpad/ui-collections` | The shell: AppShell with left sidebar, sticky header (title, breadcrumbs, actions), optional right detail panel |
| `ContentNavigation` | `@buildpad/ui-collections` | The sidebar: collection tree with search, groups, bookmarks, hidden-collection toggle |
| `CollectionList` | `@buildpad/ui-collections` | The list view: schema-aware table with search, filters, pagination, selection, bulk actions |
| `CollectionForm` | `@buildpad/ui-collections` | The detail view: dynamic create/edit form rendering every field interface |
| `useCollections` | `@buildpad/hooks` | Fetches the collection tree and manages group expand/collapse state |

<Callout type="info">
This recipe is **opt-in** ‚Äî it is *not* part of `bootstrap`. Generate the
`app/content/` module with `buildpad add content-routes` (it pulls in the
`api-routes` and `hooks` lib modules). The code below is that generated module,
shown here so you understand the wiring and can adapt it. In a generated app the
imports point at your local copies (`@/components/ui/content-layout`,
`@/lib/buildpad/hooks`); when consuming the packages directly, import from
`@buildpad/ui-collections` and `@buildpad/hooks` instead.
</Callout>

## The routes

Three files give you the whole module:

```
app/content/
‚îú‚îÄ‚îÄ layout.tsx                  # shell: sidebar + header (wraps everything below)
‚îú‚îÄ‚îÄ page.tsx                    # index: redirects to the last-used collection
‚îî‚îÄ‚îÄ [collection]/
    ‚îú‚îÄ‚îÄ page.tsx                # list view
    ‚îî‚îÄ‚îÄ [id]/page.tsx           # edit view ("+new" id = create)
```

<Steps>

### The shell layout

`app/content/layout.tsx` renders `ContentLayout` once for the whole module.
The sidebar is `ContentNavigation` fed by `useCollections`; the current
collection is derived from the URL so the active item stays in sync with
routing.

```tsx filename="app/content/layout.tsx" copy
"use client";

import { usePathname, useRouter } from 'next/navigation';
import { ContentLayout } from '@/components/ui/content-layout';
import { ContentNavigation } from '@/components/ui/content-navigation';
import { useCollections } from '@/lib/buildpad/hooks';

export default function ContentModuleLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const router = useRouter();
  const pathname = usePathname();

  // /content/[collection]/... ‚Üí active sidebar item
  const segments = pathname.split('/');
  const currentCollection = segments.length > 2 ? segments[2] : undefined;

  const {
    rootCollections,
    activeGroups,
    toggleGroup,
    showHidden,
    setShowHidden,
    hasHiddenCollections,
    showSearch,   // auto: true when >20 collections
    dense,        // auto: true when >5 collections
    loading,
  } = useCollections({ currentCollection });

  return (
    <ContentLayout
      breadcrumbs={[{ label: 'Content', href: '/content' }]}
      sidebar={
        <ContentNavigation
          rootCollections={rootCollections}
          activeGroups={activeGroups}
          onToggleGroup={toggleGroup}
          currentCollection={currentCollection}
          onNavigate={(collection) => router.push(`/content/${collection}`)}
          showHidden={showHidden}
          onToggleHidden={() => setShowHidden(!showHidden)}
          hasHiddenCollections={hasHiddenCollections}
          showSearch={showSearch}
          dense={dense}
          loading={loading}
        />
      }
    >
      {children}
    </ContentLayout>
  );
}
```

`ContentLayout` also accepts `title`, `icon`, `actions` (header buttons),
`headline`, `showBack`/`onBack`, and `sidebarDetail` for a right-hand detail
panel ‚Äî see the ContentLayout stories for each in action.

### The list view

`app/content/[collection]/page.tsx` is just `CollectionList` wired to the
router. Search, filters, pagination, column controls, and the bordered card
styling are built in.

```tsx filename="app/content/[collection]/page.tsx" copy
"use client";

import { use } from 'react';
import { useRouter } from 'next/navigation';
import { CollectionList } from '@/components/ui/collection-list';

export default function CollectionPage({
  params,
}: {
  params: Promise<{ collection: string }>;
}) {
  const { collection } = use(params);
  const router = useRouter();

  return (
    <CollectionList
      collection={collection}
      enableSearch
      enableSelection
      onItemClick={(item) => router.push(`/content/${collection}/${item.id}`)}
      onCreate={() => router.push(`/content/${collection}/+new`)}
    />
  );
}
```

Useful extras: `enableFilter` (inline filter panel), `bulkActions` (toolbar
actions for selected rows), `archiveField` (archive/unarchive UI), `fields`
(explicit column list), `tableSpacing` (`compact` / `cozy` / `comfortable`).

### The edit view

`app/content/[collection]/[id]/page.tsx` renders `CollectionForm`, which
wraps VForm with every field interface plus save/delete handling and
permission enforcement. A sentinel id (`+new`) switches it to create mode.

```tsx filename="app/content/[collection]/[id]/page.tsx" copy
"use client";

import { use } from 'react';
import { useRouter } from 'next/navigation';
import { CollectionForm } from '@/components/ui/collection-form';
import { isNewItem } from '@/lib/buildpad/utils';

export default function ItemPage({
  params,
}: {
  params: Promise<{ collection: string; id: string }>;
}) {
  const { collection, id } = use(params);
  const router = useRouter();
  const isNew = isNewItem(id);

  return (
    <CollectionForm
      collection={collection}
      id={isNew ? undefined : id}
      mode={isNew ? 'create' : 'edit'}
      onSuccess={() => router.push(`/content/${collection}`)}
      onCancel={() => router.back()}
    />
  );
}
```

## A standalone shell for your own pages

`ContentLayout` is purpose-built for schema-driven collection browsing. For
your own screens ‚Äî a dashboard, a settings page, anything that isn't the
collection browser ‚Äî `bootstrap` **wires a generic `AuthenticatedShell` into
`app/(authenticated)/layout.tsx` for you**, so every authenticated page renders
with the app chrome out of the box: a glass-blur header with a breadcrumb and
optional search/notification affordances, a sidebar with a brand mark and
navigation, and a bottom user-profile menu. The component lives at
`components/layout/AuthenticatedShell.tsx`; its styling lives in `app/globals.css`
under the `.bp-*` namespace and is built entirely from design tokens, so
light/dark mode come for free.

The generated layout renders it with **defaults** ‚Äî a single `Home` nav item and
the `NEXT_PUBLIC_APP_NAME` brand:

```tsx filename="app/(authenticated)/layout.tsx (generated)" copy
import { DaaSProviderWrapper } from '@/components/DaaSProviderWrapper';
import { AuthenticatedShell } from '@/components/layout/AuthenticatedShell';

export default function AuthenticatedLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <DaaSProviderWrapper>
      <AuthenticatedShell>{children}</AuthenticatedShell>
    </DaaSProviderWrapper>
  );
}
```

### Customizing the navigation & brand

To set your own `navItems`/`brand`, you can't pass them straight from the layout:
it's a Server Component and Tabler icons are components, which can't cross the
server/client boundary. Put the config in a small `"use client"` wrapper and
render that from the layout instead:

```tsx filename="components/PortalShell.tsx" copy
"use client";

import { AuthenticatedShell } from '@/components/layout/AuthenticatedShell';
import { IconHome, IconLayoutDashboard } from '@tabler/icons-react';
import type { ReactNode } from 'react';

const NAV_ITEMS = [
  { label: 'Home', href: '/', icon: IconHome },
  { label: 'Dashboard', href: '/dashboard', icon: IconLayoutDashboard },
];

export function PortalShell({ children }: { children: ReactNode }) {
  return (
    <AuthenticatedShell navItems={NAV_ITEMS} brand={{ name: 'Acme', initial: 'A' }}>
      {children}
    </AuthenticatedShell>
  );
}
```

```tsx filename="app/(authenticated)/layout.tsx" copy
import { DaaSProviderWrapper } from '@/components/DaaSProviderWrapper';
import { PortalShell } from '@/components/PortalShell';

export default function AuthenticatedLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <DaaSProviderWrapper>
      <PortalShell>{children}</PortalShell>
    </DaaSProviderWrapper>
  );
}
```

Props: `navItems` (sidebar links, defaults to a single `Home` entry),
`brand` (mark + name, defaults to `NEXT_PUBLIC_APP_NAME`), and
`showSearch` / `showNotifications` (presentational header affordances, both
default `true` ‚Äî wire them to your own features when ready).

The shell ships as part of the tracked
[`design-system` module](/design-system#delivery--upgrades), so
`buildpad upgrade --design` keeps the component (and its `.bp-*` styles) current.
`upgrade` refreshes the component **file**; your layout wiring and nav config stay
your code.

<Callout type="warning">
Pick one shell per route subtree. Don't nest `AuthenticatedShell` around
`/content/*` ‚Äî that already renders `ContentLayout`, and you'd get duplicate
sidebars and headers. Use `AuthenticatedShell` for your custom pages and
`ContentLayout` for the collection browser.
</Callout>

## Prerequisites

- The module must live under your authenticated layout so `DaaSProvider` is
  above it ‚Äî `buildpad init` generates this as `app/(authenticated)/` with a
  `DaaSProviderWrapper`. The data hooks call your `/api/collections`,
  `/api/fields/*`, and `/api/items/*` proxy routes.
- No extra styling is needed: the shell consumes the design tokens through
  the Mantine theme ‚Äî header and sidebar dividers use
  `--mantine-color-default-border` (slate-200), the sticky header elevates
  with `--ds-shadow-md` on scroll, and `CollectionList` renders as a bordered
  card. See the [Styleguide](/styleguide) for the token reference.
