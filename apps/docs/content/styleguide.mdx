# Styleguide

A visual reference for the design tokens. Everything on this page is rendered
from the real `app/design-tokens.css` — the swatches read the live CSS custom
properties, so this page cannot drift from the shipped values. See
[Design System](/design-system) for how the token, theme, and override layers
fit together.

The palette is built on the Tailwind reference scales: an **orange accent**
on **slate neutrals**, with Inter typography and crisp white surfaces.

<Callout type="info">
**Usage rules**

- **Main Stop**: The unsuffixed main of each hue (`--ds-primary`, `--ds-success`, …) is the **600** stop in light mode and the **400** stop in dark mode.
- **Small Text Contrast**: Small colored text on white/tinted backgrounds must use the **700** stop (`--ds-primary-700`, …) to meet WCAG AA contrast ratios. The 600 stops only pass contrast for UI components and large text.
- **Borders**: `gray-200` (`--ds-border-color`) is reserved for cards and dividers. `gray-300` is used for form inputs and boundaries.
- **Interactive Toggles**: Hover states transition to stop **700** for filled states, or to `gray-100` for outline/border states.
</Callout>

## Colors

Each hue has eleven numbered stops (50–950) plus an unsuffixed semantic main.
In dark mode the numbered scales mirror (50↔950, 100↔900, …), so a token
chosen for light backgrounds stays legible automatically. Click any swatch to
copy its variable.

## Typography

Inter for everything (`--ds-font-family`, `--ds-font-sans`, `--ds-font-display`),
JetBrains Mono for code (`--ds-font-mono`). App UI runs at the `sm` size
(`--ds-body-font-size: 0.875rem`) for density.
#
